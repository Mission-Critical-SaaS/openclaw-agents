# Scribe Knowledge Base

## Documentation Patterns
- Team prefers concise, action-oriented documentation
- Architecture decisions should follow ADR format
- Post-mortems should include: timeline, root cause, fix, prevention

## Team Context
- OpenClaw ensemble: Kit (Code), Trak (Workflow), Scout (Customers), Scribe (Knowledge)
- Primary documentation platform: Notion
- Source of truth for code: GitHub (LMNTL-AI org)
- Project tracking: Jira (lmntl.atlassian.net)

## Operational Notes
- Scribe was introduced in v1.3.56 (proactive agents phase 1)
- Budget caps and handoff protocols govern proactive operations
- Security configs (.user-tiers.json, .dangerous-actions.json) are injected at deploy time
